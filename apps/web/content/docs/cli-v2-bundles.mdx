### Usage Example

```bash
# Create SaaS Starter project
npx --registry=https://npm.pkg.github.com @xala-technologies/xaheen-cli create my-saas --preset saas-starter

# Navigate to project
cd my-saas

# Set up environment
cp .env.example .env.local
# Edit .env.local with your keys

# Run development server
npm run dev
```

## Custom Bundles

### Creating Custom Bundles

```bash
# Create bundle from services
xaheen bundle create my-stack \
  --services auth,database,payments,email \
  --description "My custom development stack"

# Create bundle interactively
xaheen bundle create my-bundle --interactive

# Export existing project as bundle
xaheen bundle export my-existing-project --output my-bundle.json
```

### Bundle Configuration Format

```json
{
  "id": "my-custom-bundle",
  "name": "my-custom-bundle",
  "displayName": "My Custom Bundle",
  "description": "Custom bundle for my specific needs",
  "version": "1.0.0",  
  "type": "custom",
  "services": [
    {
      "serviceType": "auth",
      "provider": "clerk",
      "required": true,
      "priority": 100,
      "config": {
        "features": ["organizations", "saml"]
      }
    },
    {
      "serviceType": "database", 
      "provider": "postgresql",
      "required": true,
      "priority": 90,
      "config": {
        "version": "15",
        "extensions": ["uuid-ossp", "pgcrypto"]
      }
    }
  ],
  "optionalServices": [
    {
      "serviceType": "ai",
      "provider": "openai",
      "condition": "feature:ai"
    }
  ],
  "deploymentTargets": ["vercel", "aws"],
  "compliance": ["gdpr"],
  "tags": ["custom", "saas"],
  "pricing": {
    "tier": "custom",
    "monthlyPrice": "Variable"
  }
}
```

## Bundle Best Practices

### Choosing the Right Bundle

1. **Start with your use case** - Government, SaaS, landing page, etc.
2. **Consider your timeline** - Some bundles take longer to set up
3. **Evaluate your budget** - Production costs vary significantly
4. **Think about scaling** - Some bundles handle growth better
5. **Check compliance needs** - Norwegian projects need specific bundles

### Bundle Optimization

1. **Remove unused services** after generation
2. **Upgrade services** as your needs grow
3. **Add monitoring** early in development
4. **Configure caching** for better performance
5. **Set up proper deployment** pipelines

### Common Pitfalls

1. **Over-engineering early** - Start simple, add complexity later
2. **Ignoring compliance** - Norwegian projects need specific setup
3. **Skipping monitoring** - Add observability from day one
4. **Wrong database choice** - Consider scaling and query patterns
5. **Missing backup strategy** - Plan for disaster recovery
