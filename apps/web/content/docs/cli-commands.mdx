## Commands Overview

| Command | Description |
|---------|-------------|
| `create` | Create a new Xaheen project with intelligent service bundles |
| `add` | Add services or features to existing projects |
| `remove` | Remove services from projects |
| `validate` | Check UI compliance and project health |
| `bundle` | Manage service bundles |
| `service` | Manage individual services |
| `deploy` | Deploy your application |
| `docs` | Open documentation in browser |
| `sponsors` | Display project sponsors |

## `create` - Create Project

The primary command for creating new Xaheen projects with pre-configured service bundles.

### Basic Usage

```bash
# Interactive setup (recommended)
xaheen create

# With project name
xaheen create my-project

# Use specific preset
xaheen create my-saas --preset saas-professional

# Custom bundles
xaheen create my-app --bundles auth:enterprise,billing:marketplace
```

### Service-Based Options

#### Preset Selection

| Flag | Values | Description |
|------|--------|-------------|
| `--preset` | `saas-starter`, `saas-professional`, `saas-enterprise`, `landing-page`, `marketplace`, `fintech`, `healthcare` | Pre-configured service bundles |

**Examples:**
```bash
# SaaS Starter (MVP, $0-10k MRR)
xaheen create my-mvp --preset saas-starter

# SaaS Professional (Scaling, $10k-100k MRR)
xaheen create my-saas --preset saas-professional

# Enterprise SaaS ($100k+ MRR)
xaheen create my-enterprise --preset saas-enterprise
```

#### Bundle Selection

| Flag | Values | Description |
|------|--------|-------------|
| `--bundles` | Service bundle combinations | Mix and match service bundles |

**Bundle Format:** `category:level` or bundle names

**Examples:**
```bash
# Enterprise auth with marketplace billing
xaheen create my-app --bundles auth:enterprise,billing:marketplace

# Norwegian compliance bundle
xaheen create my-gov-app --bundles norwegian-compliance

# Multiple bundles
xaheen create my-platform --bundles saas-essentials,gdpr-compliance,analytics-pro
```

### Core Service Options

#### Authentication

| Flag | Values | Default | Description |
|------|--------|---------|-------------|
| `--auth` | `better-auth`, `clerk`, `auth0`, `bankid`, `vipps`, `none` | Based on preset | Authentication provider |

**Examples:**
```bash
--auth clerk              # Clerk with enterprise features
--auth bankid            # Norwegian BankID
--auth auth0             # Auth0 with SSO/SAML
```

#### Database

| Flag | Values | Default | Description |
|------|--------|---------|-------------|
| `--database` | `postgres`, `mysql`, `mongodb`, `sqlite` | `postgres` | Database system |
| `--orm` | `drizzle`, `prisma`, `mongoose` | `drizzle` | ORM/Database toolkit |

**Examples:**
```bash
--database postgres --orm drizzle
--database mongodb --orm mongoose
```

#### Payments

| Flag | Values | Default | Description |
|------|--------|---------|-------------|
| `--payments` | `stripe`, `vipps-pay`, `paddle`, `lemonsqueezy`, `none` | Based on preset | Payment provider |

**Examples:**
```bash
--payments stripe        # Stripe with subscriptions
--payments vipps-pay    # Norwegian Vipps payments
```

### Infrastructure Options

#### Deployment

| Flag | Values | Default | Description |
|------|--------|---------|-------------|
| `--deploy` | `vercel`, `aws`, `azure`, `cloudflare`, `docker` | `vercel` | Deployment target |

#### Monitoring & Analytics

| Flag | Values | Default | Description |
|------|--------|---------|-------------|
| `--monitoring` | `sentry`, `datadog`, `newrelic` | `sentry` | Error tracking |
| `--analytics` | `posthog`, `mixpanel`, `google-analytics` | `posthog` | Product analytics |

### Compliance Options

| Flag | Values | Description |
|------|--------|-------------|
| `--compliance` | `gdpr`, `hipaa`, `pci-dss`, `soc2`, `iso27001`, `nsm` | Compliance standards |

**Examples:**
```bash
# GDPR compliant
xaheen create my-app --compliance gdpr

# Multiple compliance standards
xaheen create my-healthtech --compliance gdpr,hipaa

# Norwegian security
xaheen create my-gov-app --compliance nsm,gdpr
```

### Project Management Options

| Flag | Type | Default | Description |
|------|------|---------|-------------|
| `--git` | boolean | `true` | Initialize git repository |
| `--install` | boolean | `true` | Install dependencies |
| `--package-manager` | string | auto-detect | Package manager (npm, pnpm, bun) |

## `add` - Add Services

Add new services or features to existing Xaheen projects.

### Usage

```bash
# Add service interactively
xaheen add

# Add specific service with configuration
xaheen add billing --level enterprise

# Add compliance features
xaheen add compliance --profile gdpr

# Add authentication provider
xaheen add auth --provider bankid
```

### Service Categories

#### Authentication
```bash
xaheen add auth --provider clerk
xaheen add auth --provider bankid --features mfa,biometric
```

#### Billing & Payments
```bash
xaheen add billing --level starter      # Basic Stripe
xaheen add billing --level professional # Subscriptions + Portal
xaheen add billing --level enterprise   # Connect + Invoicing
```

#### Compliance
```bash
xaheen add compliance --profile gdpr
xaheen add compliance --profile hipaa
xaheen add compliance --profile norwegian
```

#### Infrastructure
```bash
xaheen add cache --provider redis
xaheen add queue --provider bullmq
xaheen add search --provider algolia
xaheen add storage --provider s3
```

#### Communication
```bash
xaheen add email --provider resend
xaheen add sms --provider twilio
xaheen add push --provider pusher
```

## `remove` - Remove Services

Remove services from your project safely.

### Usage

```bash
# Remove service interactively
xaheen remove

# Remove specific service
xaheen remove auth

# Remove with cleanup
xaheen remove billing --cleanup
```

### Options

| Flag | Type | Description |
|------|------|-------------|
| `--cleanup` | boolean | Remove all related code and configs |
| `--dry-run` | boolean | Preview changes without applying |

## `validate` - Validate Project

Check UI compliance, service configuration, and project health.

### Usage

```bash
# Validate entire project
xaheen validate

# Validate specific directory
xaheen validate src/components

# Generate compliance report
xaheen validate --report

# Auto-fix violations
xaheen validate --fix
```

### Options

| Flag | Type | Description |
|------|------|-------------|
| `--profile` | string | Compliance profile (minimal, standard, strict, enterprise) |
| `--fix` | boolean | Auto-fix violations where possible |
| `--report` | boolean | Generate detailed report |
| `--format` | string | Report format (markdown, html, json) |

### Validation Rules

- **UI Compliance**: Xala UI System v5 rules
- **Accessibility**: WCAG 2.2 AAA standards
- **Security**: OWASP best practices
- **Performance**: Bundle size, load time
- **Dependencies**: Version conflicts, vulnerabilities

## `bundle` - Bundle Management

Manage and explore service bundles.

### Usage

```bash
# List available bundles
xaheen bundle list

# Show bundle details
xaheen bundle info saas-professional

# Create custom bundle
xaheen bundle create my-bundle

# Extend existing bundle
xaheen bundle extend saas-starter --add ai:openai

# Compare bundles
xaheen bundle compare saas-starter saas-professional
```

### Subcommands

| Command | Description |
|---------|-------------|
| `list` | List all available bundles |
| `info <bundle>` | Show detailed bundle information |
| `create` | Create custom bundle interactively |
| `extend <bundle>` | Extend existing bundle |
| `compare <b1> <b2>` | Compare two bundles |
| `validate <file>` | Validate bundle configuration |

## `service` - Service Management

Explore and manage individual services.

### Usage

```bash
# List all services
xaheen service list

# List by category
xaheen service list --category auth

# Show service details
xaheen service info stripe

# Search services
xaheen service search "norwegian"
```

### Subcommands

| Command | Description |
|---------|-------------|
| `list` | List available services |
| `info <service>` | Show service details |
| `search <query>` | Search services |
| `compatibility <service>` | Check service compatibility |

## `deploy` - Deploy Application

Deploy your application to various platforms.

### Usage

```bash
# Deploy interactively
xaheen deploy

# Deploy to specific platform
xaheen deploy vercel
xaheen deploy aws
xaheen deploy docker
```

### Platforms

| Platform | Description |
|----------|-------------|
| `vercel` | Deploy to Vercel |
| `aws` | Deploy to AWS (EC2, ECS, Lambda) |
| `azure` | Deploy to Azure |
| `cloudflare` | Deploy to Cloudflare Workers |
| `docker` | Generate Docker configuration |
| `kubernetes` | Generate K8s manifests |

## Environment Variables

Each service comes with required environment variables. After project creation:

```bash
# Copy environment template
cp .env.example .env.local

# Required variables will be documented in .env.example
```

Common environment variables:
- `DATABASE_URL` - Database connection string
- `AUTH_SECRET` - Authentication secret key
- `STRIPE_SECRET_KEY` - Stripe API key
- `RESEND_API_KEY` - Email service key

## Troubleshooting

### Service Conflicts

If services conflict:
```bash
# Check compatibility
xaheen service compatibility clerk better-auth

# Force specific service
xaheen create my-app --auth clerk --force
```

### Bundle Issues

```bash
# Validate custom bundle
xaheen bundle validate my-bundle.json

# Reset to default bundle
xaheen create my-app --preset saas-starter --reset
```

### Validation Errors

```bash
# Get detailed error report
xaheen validate --verbose

# Fix specific rule violations
xaheen validate --fix --rule no-raw-html
```
