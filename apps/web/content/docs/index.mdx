## What is Xaheen?

Xaheen is a modern CLI tool that helps you create production-ready SaaS applications with pre-configured service bundles, enterprise-grade security, and built-in compliance. Instead of spending weeks setting up authentication, payments, and infrastructure, Xaheen gives you a complete SaaS foundation in minutes.

### Why Choose Xaheen?

#### 🚀 Service-Based Architecture
Choose business capabilities, not individual technologies. Our intelligent bundles include:
- **Authentication**: Better Auth, Clerk, Auth0, BankID
- **Payments**: Stripe, Vipps, Paddle
- **Database**: PostgreSQL, MySQL, MongoDB
- **Infrastructure**: Redis, RabbitMQ, Docker
- **Monitoring**: Sentry, DataDog, PostHog

#### 🎯 SaaS-First Design
Built specifically for SaaS applications:
- Multi-tenancy support
- Subscription billing & customer portals
- Team management & RBAC
- Admin dashboards
- Usage analytics & reporting

#### 🛡️ Enterprise Ready
Production-grade from day one:
- GDPR, HIPAA, SOC2 compliance
- SSO/SAML authentication
- Audit logging & security scanning
- Multi-region deployment
- 24/7 monitoring templates

#### 🎨 Xala UI System v5
Strict UI compliance ensures:
- WCAG AAA accessibility
- Consistent design tokens
- Built-in localization (EN, NO, FR, AR)
- RTL support
- No raw HTML elements

### Quick Example

```bash
# Install Xaheen CLI
npm install -g @xala-technologies/xaheen

# Create a new SaaS application
xaheen create my-saas --preset saas-professional

# Or use interactive mode for guided setup
xaheen create
```

## Quick Start

### 1. Install Xaheen

```bash
npm install -g @xala-technologies/xaheen
# or
bun add -g @xala-technologies/xaheen
```

### 2. Create Your Project

```bash
# Interactive mode (recommended)
xaheen create

# Or use a preset
xaheen create my-saas --preset saas-professional
```

### 3. Configure Services

```bash
# Copy environment template
cd my-saas
cp .env.example .env.local

# Add your service keys:
# - DATABASE_URL
# - AUTH_SECRET
# - STRIPE_SECRET_KEY
# - RESEND_API_KEY
```

### 4. Start Development

```bash
# Install dependencies
bun install

# Start development server
bun dev
```

Your application will be available at:
- Frontend: http://localhost:3000
- API: http://localhost:3001
